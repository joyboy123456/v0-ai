## 二、人物一致性锁定话术 ★关键反直觉发现

### 友商不显式锁定身份

3977 条 gemini3pro 案例里，**"保持脸部/保持身份/面部一致/不要改变人物"等显式锁定短语命中次数为 0**。

友商的策略不是"在 prompt 里命令模型不要改人脸"，而是：

1. **用「这套服装 / 这个服装」做隐式指代**——把"图上呈现的这套衣服"当成 prompt 中的一个固定符号，让模型在生成时把参考图当作"必须延续的视觉锚点"。
2. **不重复描述模特的脸/发型/瞳色等独立细节**——避免与参考图冲突；只描述"年龄+地域+体型"这种类别词。
3. **让人物融入场景叙事**——用"姿态/动作/神情/与道具互动"占满 prompt 主语位置，模型自然会把参考图的脸"贴上去"。

### 友商常用的"人物类别词"（不锁脸，只定调）

| 类别词 | 出现次数 |
|---|---|
| 身材高挑纤细 | 538 |
| 年轻中国男网红 | 23 |
| 年轻欧美男超模 | 14 |
| 年轻欧美男模 | 5 |
| 年轻非洲裔 | 4 |
| 年轻拉丁裔 | 1 |

骨架模板（出现极高频）：
```
一位年轻[地域][性别][身份标签]（身材高挑纤细），他/她[姿态]，穿着这套服装……
```

### 原始证据（gemini3pro.md）

> 案例 [29] ID 12368：「画面以一位年轻欧美男性为主体，在画面中央位置，视角为平视全身。人物姿态放松，右手插在裤袋，左手自然垂落，倚靠在墙面上。**他穿着这个服装**。」

> 案例 [9] ID 12646：「她以盘腿坐姿呈现，右手自然垂落，左手轻扶身旁的宠物……**模特留着黑色长发，自然垂落至背部下方**。」
> ——只在必要时一句话带过头发，不重复刻画五官。

> 案例 [10] ID 13073（猫狗系列）：「左侧这只犬穿这套服装（图4），回头看向镜头，中间这只犬穿这套服装（图6）……」
> ——多主体场景用「图N」明确指代不同参考图。

## 三、服装一致性锁定话术

### 唯一通用句式：「[主语]穿着/身着 + 这[个/套] + 服装」

完整变体：
- `他/她身着这套服装` （高频，约 609 次）
- `他/她穿着这套服装` （430 次）
- `身着这个服装` / `穿着这个服装` （200+ 次合计）
- 多张参考图：`穿这套服装（图1）`、`穿着图二服装`

**注意：** 友商几乎不写「保留服装颜色/版型/图案不变」这类反向控制。"这[个/套]服装"四个字就是"完全延续参考图"的指代。

### 多张参考图的指代约定

当一条 prompt 同时引用多个参考图时（电商场景特别常见），友商用：

> 「左侧站立着一位身着**图二服装**的年轻欧美男士……右侧的年轻欧美女士则坐在木质码头边缘的金属扶梯上，**身着图一的服装**。」（案例 [3] ID 12313）

> 「人物身着这套服装（图1）……身旁男孩穿这套服装（图2）……右侧女孩穿这套服装（图3）」（案例 [8] ID 12870 校服四人组）

**规则总结：**
- 单主体 + 单图 → `这套服装`
- 单主体 + 多图（正/背）→ 不显式说 "正面/背面"，由模型从图组中聚合（注意：友商不传细节图，他们是"纯文本生图"）
- 多主体 → `图1/图2/图3` 显式编号

### 反向控制（少数高级用法）

仅 19 次出现「不改变背景」，主要用于"在原图基础上局部改"的场景：
> 案例 [25] ID 12357：「**不改变背景。保持画面光影自然，风格一致**」（带机器人的赛博沙漠场景，作者要保护已有的沙漠 layout）
> 案例 [31] ID 12370：「**不改变背景**」（海边码头场景）

## 四、电商主图构图标准词

### 句子开头/前 30 字内必现的构图词

| 元素 | 推荐写法 |
|---|---|
| 画幅 | `竖幅构图` / `横幅构图` |
| 构图法 | `三分法` / `中央位置` |
| 镜头景别 | `全身` / `半身` / `近景胸像特写` / `中景` |
| 视角 | `平视` / `仰视` / `俯视` / `微仰拍` / `低角度俯拍视角` |
| 主体位置 | `位于画面中央偏右` / `位于画面中央偏下` / `位于画面右侧偏下` |

### 典型开头模板（实测高频）

**模板 A（最通用、最稳）：**
> 「画面采用[竖幅/横幅]构图，主体人物位于画面[中央/中央偏X]位置，以[三分法]构图原则呈现，[视角]视角，[景别]视角。」

> 案例 [1] ID 12241：「画面是一幅写实风格的摄影作品，采用竖幅构图，主体人物（年轻欧美男超模）位于画面中央偏右位置，以"三分法"构图原则呈现……」

**模板 B（开头加电影前缀）：**
> 「Arricam LT, Cooke Panchro, 50mm, f1.4。这是一张[xxx]摄影照片，采用[竖幅/横幅]构图，主体人物（[人物类别]）位于画面[位置]……」

> 案例 [3] ID 12243：「Arricam LT, Cooke Panchro, 50mm, f1.4。这是一张以清新户外为主题的摄影照片，采用横幅构图，主体人物……」

**模板 C（极简风/棚拍专用）：**
> 「[景别]，[光线]，这张图片呈现了一位[人物类别]以端庄姿态呈现，在摄影棚内的静态肖像场景。」

> 案例 [116] ID 5655：「人物近景胸像特写，暖光，这张图片呈现了一位亚裔年轻女网红以端庄姿态呈现……」

### 画幅 × 比例配合

- 竖幅 3:4 / 2:3 → 电商主图标准（友商 80%+）
- 横幅 4:3 / 3:2 → 户外、纵深场景
- 1:1 → 少数，棚拍特写

## 五、光线 & 质感修饰

### 高频光线句式

1. **柔光主流（313 次）：** "光线柔和均匀，无明显阴影"
2. **侧光（275 次）：** "光线来自左侧/右侧，呈现柔和的自然侧光"
3. **前侧光（58 次）：** "光线来自侧前方，柔和明亮"
4. **散射光（88 次）：** "呈柔和散射光，无明显强烈阴影"

### 电影前缀（点睛不是必带）

⚠️ 注意：**电影前缀出现率仅 1.5%-2.3%**，并非每条都用。多见于"户外大片/海边帆船/古典建筑"等需要质感拉满的场景，棚拍纯背景几乎不用。

**两个固定句式：**
1. `Arricam LT, Cooke Panchro, 50mm, f1.4。` （Arri 电影机 + Cooke 镜头 + 大光圈）
2. `IMAX Film Camera Arri Signature Prime, 24mm f11。` （IMAX 胶片 + 24mm 广角 + 小光圈）

**位置：** 必须在 prompt **最开头**，单独成句，以句号结尾。

### 质感后缀（高频组合）

```
具有电影感的逼真渲染效果
时尚杂志的经典大片风格
写实风格
细节丰富，[xxx]纹理清晰
画面质感细腻
8K 商业画质（极少，仅6次）
```

句子里的典型组合（案例 [1] 完整结尾）：
> 「整体亮度适中，**细节丰富**，轿车的金属轮毂、建筑的窗户排列等都清晰可见，**具有电影感的逼真渲染效果**，整体氛围宁静而富有格调，仿佛在捕捉一段复古优雅的城市时光，让人联想到**时尚杂志的经典大片风格**……」

### 场景描述长度统计

抽样观察：典型 gemini3pro 案例 prompt 总长 **200-450 中文字**（接近 800 字符 API 上限），其中：
- 构图+视角+人物：约 80-120 字
- 姿态描述：60-120 字
- 场景描述：80-150 字（核心，决定氛围）
- 光线+质感：60-100 字

**先环境后细节** 与 **先人物后环境** 并存，但 80% 案例采用：**构图开头 → 人物 → 姿态 → 服装锚点 → 场景 → 光线 → 后缀**。

## 七、防崩坏话术

### 真相：友商不写显式 negative

如第二章统计所示，"保持脸部 / 不要修改 / 无畸形 / 颜色不变" 这类话术在 3977 条 gemini3pro 案例里 **基本是 0 命中**。

### 友商防崩的策略（隐式）

1. **占位符代替反向描述：** 用「这套服装」让模型把参考图当锚点，不再额外发挥。
2. **写满正向细节：** prompt 写满 400 字的姿态、场景、光线，模型没有"自由空间"乱发挥。
3. **明确指代多元素：** 用「这只犬 / 这个包 / 这副墨镜 / 这副耳环」给每个参考物体单独的"这"指代，避免模型把它们合并改造。
4. **极少数显式：**

   - `不要出现文字` （在棚拍套图中出现，案例 [9] ID 12646）

## 八、与我们当前 buildPoseFissionPrompt 的 diff

### 我们当前的 prompt（`lib/server/pose-fission-service.ts:117-138`）

```ts
return [
  '基于上传的服装模特成片进行姿势裂变。',
  '第一张图是需要保持人物、服装和画面质感的主图，后续图片若存在则是产品正面或背面细节参考。',
  `目标姿势：${template.name}。`,
  template.prompt ? `姿势要求：${template.prompt}。` : '',
  params.hasFrontDetail
    ? '已提供产品正面细节图，请保持领口、面料、logo、图案等正面细节一致。'
    : '',
  params.hasBackDetail
    ? '已提供产品背面细节图，请在背面或侧后角度中保持背部结构一致。'
    : '',
  `画面比例：${params.imageRatio}。`,
  `分辨率档位：${params.resolution}。`,
  '要求：保持原图人物身份、脸部特征、发型、身材比例、服装颜色、版型、材质、图案和关键细节；只改变人物姿势和必要构图；生成电商主图质感，背景干净，主体清晰；避免手部畸形、服装扭曲、肢体异常、脸部崩坏、文字乱码和多余人物。',
]
```

### 缺失：我们 **没有** 但友商 **常用** 的关键短语

1. **占位符指代语言**——我们说"原图人物""第一张图"，友商说"这套服装/这个服装"。
   - 影响：Gemini 在多模态推理时，对"原图"这种**指令性指代**响应不如对"这个/这套"**实体性指代**精准。
2. **构图开头的标准句式**——我们没写「竖幅构图，主体位于…，平视全身视角，三分法」。
3. **人物类别词**——我们没给"年龄+地域+身份+体型"。
4. **场景叙事**——我们的 prompt 只说"背景干净，主体清晰"，没有营造真实场景的细节。
5. **光线/氛围**——完全缺失，连"柔光"两个字都没有。
6. **质感后缀**——没有"写实风格 / 时尚杂志大片 / 细节丰富 / 电影感"等点睛词。
7. **画幅与比例的语义化表达**——我们直接给 `params.imageRatio` 字符串（如 `3:4`），友商写"竖幅构图"。模型对自然语言比对原始比例标识更敏感。

### 写得不如友商好的部分

1. **显式反向控制堆叠**——
   > 「避免手部畸形、服装扭曲、肢体异常、脸部崩坏、文字乱码和多余人物」
   是大量 SDXL/MidJourney 时代的 "negative prompt 思维"。**Gemini 3 Pro 对反向描述的响应不稳定**，且把"畸形/崩坏"这类词放进 prompt 反而可能让模型激活相关概念（实际案例中友商基本不写这类词）。
2. **"保持xxx、保持yyy、保持zzz"列表**——
   把人物身份、脸、发型、服装五个维度全列出来。友商用一个「这套服装」+ 不重复刻画脸部细节，反而隐含锁定。我们的列表会让模型在每个维度都"自由发挥"取舍。

### 已经做对的部分（不动）

1. ✅ "基于上传的服装模特成片"——明确告诉模型这是图生图（虽然 Gemini 通过图片附件已经知道，但显式说更稳）。
2. ✅ 把姿势模板 prompt 串进来——这正是友商「postureLibId」预设库的等价物。
3. ✅ 把比例/分辨率传给底层 adapter——和友商参数对齐。
4. ✅ 失败容忍 + 流式持久化（pipeline 层）。

## 九、落地建议（3 套 prompt 模板雏形）

> 以下模板均为**字符串 prompt**，给开发者直接替换 `buildPoseFissionPrompt` 的 return 字符串。
> 占位符语义：`{poseName}`=`template.name`，`{posePrompt}`=`template.prompt`，`{ratio}`=如 `3:4`，`{ratioWord}`=`竖幅构图`/`横幅构图`/`方图`。

### 方案 A：「极简对齐友商」——最小改动，最大效果（推荐先试）

```
这是一张写实风格的电商服装大片摄影作品，采用{ratioWord}，
主体人物（参考第一张图的同一个人，身材比例与发型保持一致）
位于画面中央，以平视全身视角呈现，以"三分法"构图原则布局。

人物穿着这套服装（图1为主图基准；{frontHint}{backHint}），
姿态调整为：{poseName}。{posePromptLine}

背景干净简洁，符合电商主图调性，主体清晰突出。
光线为柔和均匀的自然光，无强烈硬阴影，光线统一。
画面色彩饱和度适中，细节丰富，衣物的纹理、面料质感、
图案与 logo 清晰可见，具有电影感的逼真渲染效果，
整体氛围接近时尚杂志的经典大片风格。

不改变服装的颜色、版型、材质、图案与 logo；
不改变人物的脸部特征与发型；保持画面风格一致。
```

其中：
- `{frontHint}` = `图2为正面细节参考` （仅当 `hasFrontDetail`）
- `{backHint}` = `图3为背面细节参考` （仅当 `hasBackDetail`）
- `{posePromptLine}` = `${posePrompt}。` 或空字符串
- `{ratioWord}` 映射：
  - `3:4` / `2:3` / `9:16` → `竖幅构图`
  - `4:3` / `3:2` / `16:9` → `横幅构图`
  - `1:1` → `方图构图`

**关键改动：**
1. 引入「这套服装」占位符（学友商）
2. 把 ratio 转成「竖幅/横幅」自然语言
3. 把"保持xxx"压缩成 2 句结尾（保留少量反向控制，但不堆叠）
4. 加质感后缀（电影感 + 时尚杂志大片）
5. 保留多图编号（图1/图2/图3）

### 方案 B：「场景化加强」——传姿势 + 场景模板（适合后续接入场景库）

```
{cinematicPrefix}{ratioWord}的电商服装大片。

画面主体是一位（参考第一张主图的同一个人，身材高挑纤细，
保持发型、脸部特征与肤色一致），位于画面中央偏{positionHint}位置，
采用全身平视视角，以"三分法"构图呈现。

人物穿着这套服装（{detailRefs}），姿态调整为：{poseName}。
{posePromptDetail}

{sceneDescription}

光线柔和明亮，{lightDirection}，无明显强烈阴影，
整体色调柔和，色彩饱和度适中，传递出{moodWord}的情绪。
细节丰富，衣物纹理、面料质感、图案与 logo 清晰可见，
具有电影感的逼真渲染效果，呈现时尚杂志大片风格。

保持原图服装的颜色、版型、材质、图案不变；保持人物身份与发型不变。
```

字段说明（这些字段在 `PoseTemplate` 或新增「sceneId」配置里维护）：
- `cinematicPrefix`：`Arricam LT, Cooke Panchro, 50mm, f1.4。` 或空（按场景需要）
- `positionHint`：`右` / `下` / `左`
- `detailRefs`：`图1为主图，图2为正面细节，图3为背面细节` 之类的动态字符串
- `sceneDescription`：从场景模板库取的一段背景描述（"背景是欧洲风格的沿河建筑群……"）
- `lightDirection`：`来自画面左侧的侧光` / `前侧光` / `散射光`
- `moodWord`：`宁静优雅` / `清新活力` / `轻松惬意` / `复古都市`

这一版**完全对齐友商黄金公式**，留出未来"场景库"的接口。

### 方案 C：「双语 + 防御性 + 显式锚定」——保守稳定派

```
[Editorial fashion lookbook, photorealistic, {ratioWord}, three-thirds composition.]

参考主图：第一张图（图1）是必须延续的人物 + 服装锚点。
{frontDetailLine}{backDetailLine}

任务：基于图1的人物与服装，生成同一个人、同一套服装、
不同姿势与构图的电商主图。新姿势为：{poseName}。
{posePromptLine}

人物呈现：位于画面中央，平视全身视角，三分法构图。
姿态自然放松，神情自然。

服装呈现：完全延续图1服装的颜色、版型、材质、图案、纹理、
logo 与配饰；不增加、不减少、不修改任何服装元素。

人物呈现：保持图1人物的脸部特征、发型、肤色、身材比例完全一致。

场景：背景简洁干净，浅色或单色调摄影棚风格，符合电商主图调性，
主体清晰突出。光线为柔和均匀的影棚灯光，无强烈明暗对比，
阴影柔和过渡，整体色调干净通透。

质感：写实风格摄影，画面细节丰富，衣物纹理与面料质感清晰可见，
具有电影感的逼真渲染效果，时尚杂志大片质感。

不要出现：多余人物、文字、水印、手部畸形、服装扭曲、五官变形。
```

其中：
- `frontDetailLine` = `图2为该服装的正面细节参考，请保持领口、扣件、logo、图案细节与图2一致。` （仅当 `hasFrontDetail`）
- `backDetailLine` = `图3为该服装的背面细节参考，请在背面或侧后角度中保持背部结构、印花、剪裁与图3一致。` （仅当 `hasBackDetail`）

这版**保留显式反向控制**，适合 risk-averse 路线。可以作为 A/B 测试中的"对照组"。

### 3 套方案对比

| 维度 | A 极简对齐 | B 场景化 | C 防御性 |
|---|---|---|---|
| 字数 | 200 字 | 300 字 | 350 字 |
| 友商对齐度 | 高 | 极高 | 中 |
| 一致性锁定 | 隐式 + 1 句显式 | 隐式 + 2 句显式 | 显式（全列出） |
| 场景描述 | 无 | 有（需场景库） | 简单棚拍 |
| 反向控制 | 弱（2 行） | 弱（2 行） | 强（全列） |
| 实现复杂度 | 改 20 行 | 改 + 加场景库 | 改 30 行 |
| 推荐使用顺序 | 1️⃣ 先试 | 2️⃣ 接入场景后 | 3️⃣ 对照组 |

### 实现建议

1. **第 1 步：** 用方案 A 替换现有 `buildPoseFissionPrompt`，跑 1 组同人同衣 9 pose 看一致性是否提升。
2. **第 2 步：** 把 ratio→ratioWord 的映射抽成 `lib/server/pose-fission-service.ts` 内部小函数（已有 `params.imageRatio`，加 12 行映射即可）。
3. **第 3 步：** 若一致性已显著改善，考虑下版本引入"场景模板"（对齐友商 `postureLibId` 思路），用方案 B。
4. **第 4 步：** A/B 测试用方案 C 做对照组，验证"显式反向控制"对 Gemini 3 Pro 是否真的有副作用。

### 风险提示

- 友商 prompt 是「文本生图」，我们是「图生图」（带参考图），Gemini 3 Pro 在两种模式下对"这套服装"占位符的解释可能不同——需要实测验证。
- 友商不传"正面/背面细节图"（他们只有 1 张参考图），我们的多图场景需要补充编号说明（方案 A/B/C 都做了，但具体编号语序要在第一次跑通后微调）。
- 「不改变背景」是友商保护参考图背景用的，姿势裂变恰恰要换姿势但保留人物服装，因此 **不要在我们的 prompt 里加「不改变背景」**——我们的逻辑是"换姿势可能换构图，但不换人和衣"。
